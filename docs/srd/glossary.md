# 术语表

## 领域术语
| 术语 | 定义 |
|---|---|
| 预测市场 | 用户买卖对某事件可能结果的敞口的市场；价格代表隐含概率。 |
| 市场 | 针对某一事件和问题的可交易预测实例（例如，“谁会赢？”）。 |
| 选项（结果选项） | 市场中可选择的结果（例如，A队，平局，B队）。 |
| 份额/持仓 | 用户下注/交易后持有的某选项的敞口。 |
| 流动性 | 市场/投注中可用于支持交易和定价的可用资金。 |
| 总流动性（市场） | 市场上所有已配置的投注的总流动性（用于庄家创建流程）。 |
| 庄家模式 | 市场创建者提供初始流动性并根据既定规则接受用户投注的模式。 |
| 隐含概率 | 从当前市场价格推导出的概率；在本软件需求文档中，显示为百分比（“胜率/份额”）。 |
| 十进制赔率 (oddsDecimal) | 用于表示某选项回报倍率的数值（>1）；本 SRD 中“潜在回报”计算采用 `本金 * oddsDecimal`。 |
| 潜在回报 | 如果所选选项获胜，用户可能收到的总金额；本 SRD 中默认根据本金和 `oddsDecimal` 计算。 |
| 费用 | 应用于投注/交易的平台收费（例如，0.2%）。 |
| 截止时间（关盘时间） | 不允许下新注之后的时间点。 |
| 结算 | 结果确定后市场的最终确定；持仓清算并应用支付。 |
| 裁决 | 宣布获胜选项和任何特殊裁定的过程/结果。 |
| 挑战期 | 在候选结果提交后允许任何人提出异议的时间窗口；若无异议，候选结果可被视为最终结果。 |
| UMA Protocol | 本版本裁决集成基线中的预言机提供方（乐观预言机）。 |
| Kleros Court | 本版本裁决集成基线中的争议仲裁提供方（去中心化法庭）。 |
| 精选/置顶 | 运营者配置的优先处理，用于可见性和排序。 |

## 用户体验术语
| 术语 | 定义 |
|---|---|
| 骨架屏 | 数据加载时使用的占位符用户界面。 |
| 空状态 | 没有数据时显示的界面（例如，没有可用事件）。 |
| 提示信息 | 向用户显示的短暂通知消息。 |

## 缩写
| 缩写 | 含义 |
|---|---|
| SRD | 软件需求文档 (Software Requirements Document) |
| PRD | 产品需求文档 (Product Requirements Document) |
| UI | 用户界面 (User Interface) |
| NFR | 非功能性需求 (Non-Functional Requirement) |
| FR | 功能性需求 (Functional Requirement) |
