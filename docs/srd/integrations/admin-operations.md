# 集成 — 管理员/运营配置提供商

## 1. 目的
提供由运营人员管理的配置，用于发现和创建流程（特色/固定的市场、投注配置、预设、费率）。

## 2. 配置域
- 特色市场列表及其排序
- 市场列表排序的固定权重
- 启用的体育项目/联赛和可见性控制
- 每个体育项目的投注类型可用性
- 每个体育项目的趣味投注定义
- 费率配置

## 3. 接口要求
- 面向客户端的 API 必须通过 `api-specifications/operations.md` 暴露这些值。
- 配置更改必须可审计（谁/何时/更改了什么）。
- 配置应版本化，并通过 `version` 字段可检索。

## 4. 故障模式和后备方案
- 如果配置提供商不可用，客户端将回退到安全默认值：
  - 没有特色市场
  - 固定权重 = 0
  - 费率默认值为 0.2%
