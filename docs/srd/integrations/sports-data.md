# 集成 — 体育数据

## 1. 目的
提供权威的体育分类（运动/联赛）、赛事日程、参与者（队伍/球员）和开始时间，供发现和市场创建使用。

## 2. 数据交换
### 输入到本系统
- 体育类别和启用状态
- 每个运动的联赛
- 每个联赛的赛事
- 每个赛事的参与者（队伍、球员）
- 赛事开始时间

### 输出（返回提供方）
- 本 SRD 无要求

## 3. 接口要求
- 面向客户端的 API 必须通过 `api-specifications/events.md` 暴露体育数据。
- 赛事开始时间必须一致并包含时区（推荐使用 UTC）。
- 冠军赛式赛事必须表示为 `eventType=CHAMPION` 且具有稳定的 `eventId`。

## 4. 故障模式和回退机制
- 如果体育数据不可用：
  - 发现功能应显示错误/空状态，但仍可导航。
  - 市场创建应阻止赛事选择，并提供重试机制。

## 5. 数据质量约束
- 面对面赛事（head-to-head）的参与者必须恰好包含两个主要参与者。
- 参与者名称必须准备好显示且稳定。
