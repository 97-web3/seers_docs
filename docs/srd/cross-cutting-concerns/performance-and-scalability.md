# 性能和可伸缩性

## 1. 性能要求（最低目标）
- 发现列表：首个有意义的内容应立即以骨架屏形式出现；数据应快速填充（目标阈值待最终确定）。
- 市场详情：即使图表系列加载缓慢，期权也必须可用；图表应逐步加载。
- 交易提交：提交反馈（加载状态）必须立即出现，以防止重复点击。

## 2. 可伸缩性考虑因素
- 热门比赛期间预计出现峰值负载；系统必须扩展读取端点（市场、期权、价格历史记录）。
- 写入端点（投注、市场创建）在并发情况下必须保持正确性。

## 3. 缓存指导（行为性，非实现性）
- 配置响应可以具有较短的 TTL 进行缓存，并通过版本更改使其失效。
- 市场列表可以短暂缓存，但对交易至关重要的值（流动性、份额）应频繁刷新。
