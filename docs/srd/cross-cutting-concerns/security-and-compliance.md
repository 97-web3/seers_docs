# 安全与合规

## 1. 安全要求
- 所有经过身份验证的端点必须强制执行访问控制。
- 敏感数据（令牌、钱包余额、用户标识符）在传输过程中必须受到保护（HTTPS），并且不得以纯文本形式暴露在日志中（策略）。
- 改变状态的操作必须通过幂等性密钥来防止重放/重复提交。

## 2. 财务完整性
- 下注和市场创建的资金必须是原子化和可审计的。
- 系统必须防止因竞态条件导致的负余额。

## 3. 隐私
- 用户只能查看自己的持仓和下注历史。
- 选项列表中显示的参与者头像预览不得泄露超出预期的私有标识符（例如，如果需要，使用匿名头像/别名）。

## 4. 合规性
- 法规要求（如果有）不在本 PRD 派生的 SRD 的范围之内，必须由利益相关者提供；系统在设计时必须能够支持未来的限制（例如，年龄限制、管辖权限制），而不会破坏核心契约。
