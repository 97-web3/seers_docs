# 可观测性

## 1. 目标
- 检测体育数据、钱包和结算依赖项的停机。
- 端到端追踪和调试改变状态的操作（投注、市场创建）。

## 2. 所需遥测（行为性）
- 所有 API 请求和响应的结构化日志（根据策略排除敏感载荷数据）。
- 关联 ID：
  - 接受客户端提供的请求 ID，或在服务器端生成一个。
  - 在错误信封中返回 `requestId`。
- 指标：
  - 按端点组（发现/详情/交易/创建）划分的请求率、错误率、延迟。
  - 按错误代码划分的投注放置成功/失败计数。
  - 按验证类别划分的市场创建成功/失败计数。

## 3. 告警（最低要求）
- 钱包依赖项停机
- 投注放置失败率升高
- 在高峰时段市场列表失败率升高
